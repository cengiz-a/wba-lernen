Einbindung über eine externe Datei

mit link-Element

<!doctype html>
<html>
  <head>
    <meta charset="utf-8">
    <title>Stylesheet über externe Datei einbinden</title>
    <link rel="stylesheet" href="stylesheet.css" type="text/css">
    <link rel="stylesheet" href="../css/reveal.css" type="text/css" >  
  </head>

  </body>

  • Stylesheet kann von mehreren Dokumenten eingebunden werden
  • Es können mehrere Stylsheets in einem Dokument eingebunden werden

Eine bestimmte Klasse darf beliebig vielen Elementen in einem Dokument zugewiesen werden. Eine bestimmte ID darf nur einem einzigen Element in einem Dokument zugewiesen werden.

Einfache Selektoren

Einfache Selektoren können auch miteinander kombiniert werden.

p, div {…} Alle p- und div-Elemente
p.info {…} Alle p-Elemente mit der Klasse "info"
.info, .wichtig {…}   Alle Elmente mit der Klasse "info" oder"wichtig"
.info.wichtig {…} Alle Elemente mit den Klassen "info" und "wichtig"
.info .wichtig Alle Elemente die der Klasse "wichtig" angehören und sich innerhalb der Elemente mit der Klasse "info" befinden>

Attribut-Selektoren

[href] {…} Alle Elemente mit dem Attributnamen "href"
[href=http://google.de] {…} Alle Elemente deren Attribut "href" exakt den Wert "http://google.de" haben
[href*=google] {…} Alle Elemente die das Attribut "href" haben und deren Wert die Zeichenkette "google" enthalten
[href^=http] {…} Alle Elemente die das Attribut "href" haben und deren Wert mit der Zeichenkette "http" beginnen
[href$=http] {…} Alle Elemente die das Attribut "href" haben und deren Wert mit der Zeichenkette "http" enden

Block- vs. Inline-Elemente

Block-Elemente…

  • sind so breit wie ihr Elternelement, falls kein anderer Wert angegeben wird
  • sind so hoch wie ihr Inhalt, falls kein andererr Wert angegeben wird
  • können Innen- und und Außenabstände haben
  • erzeugen einen Umbruch
  • sind u.a. <p>, <div>, <form>, <header>, <nav>, <ul>, <li>, und <h1>

Inline-Elemente…

  • ignorieren Höhen- und Breitenangaben
  • befinden sich im Textfluss
  • können Abstände nach oben und unten, jedoch nicht nach links und rechts, zugewiesen werden
  • erzeugen keinen Umbruch
  • sind u.a. <a>, <span>, <em>, <cite>, und <code>

Block- vs. Inline-Elemente

Über die CSS-Eigenschaft 'display' kann die Anzeigeart eines Elements geändert werden

p { display: none } Element wird nicht angezeigt und es wird auch kein Platzhalter freigelassen.
span { display: block } Erzwingt einen Block - das Element erzeugt eine neue Zeile.
p { display: inline } Erzwingt die Anzeige im Text - das Element wird im laufenden Textfluss angezeigt.
span { display: inline-block } Erzeugt äußerlich einen Block, für den Breite, Höhe und Außenabstand angegeben werden kann, belässt das Element jedoch im laufenden Textfluss.

Das Box-Modell

Im visuellen Anzeigemodell erfolgt die Darstellung von Elementen durch das Zeichnen von Rechtecken. Die Bestandteile dieser Rechtecke beschreibt das „Box-Modell“.

Quelle: SELFHTML-Wiki - CSS/Box-Modell (aufgerufen am 13.11.2014)

Das Box-Modell

Quelle: Dr. Web - CSS/Box-Modell (aufgerufen am 13.11.2014)

Das Box-Modell

  • Für Block-Elemente können Höhe und Breite beliebig festgelegt weren
  • Bei Inline-Elementen werden die Maße durch den Inhalt vorgegeben
  • über die Eigenschaft display: block kann ein Inline-Element als Block-Element dargestellt werden
  • Hintergrundbilder- und farben werden in den Bereich des Inhalts, Innenabstands und Rahmen gezeichnet.
  • Die Gesamtbreite eines Elements errechnet sich durch: 
    margin-left + border-left + padding-left + width + padding-right + border-right + margin-right
  • Die Gesamthöhe eines Elements sich durch: 
    margin-top + border-top + padding-top + height + padding-bottom + border-bottom + margin-bottom

Alternatives Box-Modell

  • Problem des klassischen Box-Modell: Höhen- und Breitenangabe sind ungenau, da sie sich nur auf den Content beziehen
  • Die Deklaration box-sizing: border-box aktiviert ein alternatives Boxmodell
  • Padding und Border wird in die Höhen- und Breitenangabe mit einbezogen

Innen- und Außenabstände

Die Abstände können für jede Seite einzeln definiert weren

p {
	margin-top: 10px;
	margin-right: 20px;
	margin-bottom: 30px;
	margin-left: 5px;
}

oder in Kurzschreibweise definiert werden

/* Abstände werden im Uhrzeiger definiert, beginnend mit der oberen Seite */
p { margin: 10px 20px 30px 5px;} 
					
/*  für alle 4 Abstände gilt derselbe Wert */
p {margin: 20px; }

/* 1. Wert für margin-top und margin bottom. 
   2. Wert für margin-left und margin-right */
p { margin: 10px 20px; } 
			
/* Setzt den rechten und linken Abstand auf die noch 
   zur Verfügung stehende Breite */
p { margin: 0 auto; }

Farben über Hexadezimal-Werte

p {background-color: #228B22;}
  • Drei Pärchen aus Hexadezimalwerten
  • Ein Pärchen besteht aus zwei Hexadezimalziffern
  • Die drei Pärchen stehen für die Farbwerte Rot, Grün und Blau. Genau in dieser Reihenfolge
  • In dem Beispiel: Rot: 22 – Grün: 8B – Blau: 22

Farben über RGB-Werte

Quelle: wikimedia.org (aufgerufen am 14. Oktober 2014)

Farben über RGB-Werte

  • Farbe wird durch die Addition von Rot, Grün und Blau gebildet
  • Mit prozentualen Angaben zwischen 0% und 100% 
    p {background-color: rgb(13.33%,54.51%,13.33%);}
  • Mit Werten zwischen 0 und 255 
    p {background-color: rgb(34,139,34);}
  • Über einen vierten Wert kann eine Transparenz zwischen 0 und 1 definiert werden 
    p {background-color: rgba(34,139,34, 0.5);}

Farben über HSL-Werte

Quelle: wikimedia.org (aufgerufen am 14. Oktober 2014)

Farben über HSL-Werte

  • Farbe wird durch den Farbwinkel (H) im Farbkreis, der Sättigung (S) in Prozent und Helligkeitswert (L) in Prozent gebildet
  • Mit prozentualen Angaben zwischen 0% und 100% 
    p {background-color: hsl(120,61%,34%);}
  • Über einen vierten Wert kann eine Transparenz zwischen 0 und 1 definiert werden 
    p {background-color: hsla(120,61%,34%, 0.5);}

Farben über vorderfinierte Schlüsselworte
