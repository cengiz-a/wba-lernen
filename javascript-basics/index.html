Was ist Javascript? Eine Einführung

  • Eine Skriptsprache
  • Unterschied zwischen Skript- und Programmiersprache?
  • Gängige Webbrowser können Javascript interpretieren und den Code ausführen (Lynx leider nicht)
  • Javascript lässt sich in HTML Dokumenten einfach integrieren und kann dessen Struktur und aussehen zur Laufzeit verändern
  • Wird sowohl Serverseitig als auch Clientseitig verwendet (eine Sprache für alles, Serverseitiges Rendern - ReactJS (Universal), Schneller, Client baut die Seite nicht im Browser)
  • Funktional und (beschränkt) Objektorientiert - Prototypen Basiert

Folien basieren auf dem EcmaScript5 Standard (Release 2011, 6 & 7 sind WIP)

ECMA = European Computer Manufacturers Association (u.a. Microsoft)

Entwickelt von Brendan Eich (Netscape), am Anfang hiess es noch Mocha, LiveScript

Die häufigsten Syntaxfehler

  • Fehlen schließende runde oder geschweifte Klammern?
  • Fehlt ein schließendes Hochkamma oder wird ein doppeltes Hochkomma durch ein einfaches Hochkomma geschlossen?
  • Klammer zuviel im Script-Code?
  • Bindestrich im Variablennamen? Javascript mag keine Bindestriche.
  • Groß und Kleinschreibung beachtet? Hier ist Javascript tatsächlich empfindlich
  • Schreibfehler in einer der DOM-Methoden? z.B.: getElementById vs getElementbyId.

Korrekte Variablennamen

  • bestehen aus Buchstaben (wobei auch ein Unterstrich „_“ als Buchstabe zählt),
  • aus Ziffern,
  • dürfen keine Leerzeichen enthalten,
  • dürfen nicht mit einer Zahl, einem Sonderzeichen oder Satzzeichen anfangen und
  • dürfen nicht mit einem Schlüsselwort übereinstimmen.

Funktionen

  • Funktionen sind ein Block von Anweisungen mit einem Namen.
  • Der Funktionsblock wird einmal definiert.
  • Dann können die Javascript-Befehle der Funktion über den Namen mehrfach im Programm aufgerufen werden.
  • Das Zusammenfassen von Befehlen in einer Funktion verbessert obendrein die Lesbarkeit des Scripts.

              
                    +--- Schlüsselwort
                    |       +---> Name der Funktion
                    |       |
                    |       |      +---> Argument
                    |       |      |       +---> Argument
                    |       |      |       |
                function steuer ( betrag, satz) {
                   var mwst = betrag * satz / 100;        // <-+
                   var summe = betrag + mwst;             //   +  Anweisungsblock
                   return summe;                          // <-+
                }
                …

                var brutto = steuer ( 200, 16 );          // Funktionsaufruf

Objekte

  • Objekte sind Behälter für logisch zusammen gehörende Variablen und Funktionen
  • Objekte können bestimmte Eigenschaften und Methoden zugewiesen werden

  • Wer von den formalen Programmiersprachen kommt, findet Javascript-Objekte recht simpel. Javascript hat ein beschränktes Objekt-Modell.

DOM und Vererbung

Document Object Model

Document Object Model (DOM) ist eine Spezifikation einer Schnittstelle für den Zugriff auf HTML- oder XML-Dokumente. Sie wird vom World Wide Web Consortium definiert.

Quelle: Wikipedia - Document Object Model (aufgerufen am 15.10.2014)

W3C - Document Object Model Specifications

DOM und Vererbung

<html>
    <head>
      <title>DOM</title>
    </head>
    <body>
      <p>Lorem Ipsum</p>
      <ul>
        <li>Apfel</li>
        <li>Birne</li>
        <li>Kirsche</li>
      </ul>
    </body>
  </html>
  • document ist das Wurzelelement
  • head und body sind Geschwister
  • ul und li sind Eltern- und Kind-Element

DOM und Javascript

Mit Javascript kann man über den DOM Baum navigieren und HTML Tags, Attribute oder Inhalte verändern, hinzufügen oder entfernen
  • DOM zugreifen getElementsByTagName()
  • Neue Knoten erzeugen createElement()
  • HTML Attribute zugreifen und manipulieren getAttribute()
  • Durch den DOM Baum navigieren parentNode
  • Den DOM Baum manipulieren insertBefore()
  • Den DOM Baum analysieren nodeName

BOM

  • The Browser Object Model
  • Kein Offizieller Standard
  • Dient zur Manipulation mit dem Browserfenster
  • navigator, history, screen, location and document sind children von window
  • document repräsentiert dann den Inhalt der Seite - den DOM
  • http://fiddle.jshell.net/koos/hdtg0a9f/

Einbettung und Zusammenhang von Javascript und HTML

  • HTML - Hypertext Markup Language / CSS - Cascading Style Sheets
  • NEU: Javascript
  • Einbettung von Javascript in 2 Formen möglich
    • Code direkt im script Tag innerhalb des head oder head Tags
    • Code verlinkt im script Tag innerhalb des body Tags
  • Praxis: alles an einem Ort
  • Und wenn es geht am Ende der HTML Datei

Einbettung von Javascript Code im Body

             
               <!DOCTYPE html>
               <html>
               <body> 

               <h1>My Web Page</h1>

               <p id="demo">A Paragraph</p>

               <button type="button" onclick="myFunction()">Try it</button>

               <script>
               function myFunction() {
                  document.getElementById("demo").innerHTML = "Paragraph changed.";
               }
               </script>

               </body>
               </html>

Einbettung von Javascript Code im Head

             
               <!DOCTYPE html>
               <html>
               <head>
               <script>
               function myFunction() {
                  document.getElementById("demo").innerHTML = "Paragraph changed.";
               }
               </script>
               </head>
               <body> 

               <h1>My Web Page</h1>

               <p id="demo">A Paragraph</p>

               <button type="button" onclick="myFunction()">Try it</button>

               </body>
               </html>

Einbettung einer externen Javascript Datei im Body/Head

             
              <!DOCTYPE html>
              <html>
              <body>
              <script src="myScript.js"></script>
              </body>
              </html>
