Positionierung von Elementen

  • static (default)
  • relative
  • absolute
  • fixed
  • (sticky)

position: static

  • Default für alle Elemente
  • top, right, bottom, left, z-index ohne Auswirkungen
  • Inhalte erscheinen in der Reihenfolge, wie sie im Quelltext stehen
.blaue_box {
            position: static;
            bottom: 20px; /*Das würde nichts bewirken*/
            left: 15%;    /*Auch überflüssig*/
        }

position: relative

  • Grundsätzlich wie static
  • Versatzoptionen top, right, bottom, left, z-index sind verfügbar
  • Inhalte erscheinen in der Reihenfolge, wie sie im Quelltext stehen
.blaue_box {
            position: relative;
            bottom: 20px;
            left: 15%;
        }

position: absolute

  • Element wird aus dem normal flow entfernt
  • Versatzoptionen top, right, bottom, left, z-index sind verfügbar
.blaue_box {
            position: absolute;
            bottom: 20px;
            left: 15%;
        }

position: fixed

  • Element wird aus dem normal flow entfernt
  • Versatzoptionen top, right, bottom, left, z-index sind verfügbar
  • Koordinatensystem jedoch immer gemessen am "Viewport" (Browserfenster, Bildschirm mobiler Geräte)
  • Beispiel
.header {
            position: fixed;
            top: 0px;
        }

position: sticky

  • Element wird aus dem normal flow entfernt
  • Versatzoptionen top, right, bottom, left, z-index sind verfügbar
  • Bleibt solange kleben bis der übergeordnete Container aus dem Viewport ist
  • Beispiel (im FF oder Safari anschauen)
.header {
            position: sticky;
            top: 10px;
        }
Fragen zur Positionierung?

Der z-Index

Was wenn Elemente übereinander liegen?

  • Zuletz genanntes Element im HTML Code liegt vorne

.rot {
    z-index: 2;
}

.blau {
    z-index: 3;
}

.gruen {
    z-index: 4;
}
  • Element mit höherem z-index überdeckt Elemente mit niedrigerem z-index

Dokumentenfluss

  • CSS Eigenschaften float und clear
  • CSS Eigenschaften float und clear
  • Float positioniert die Elemente im normal Flow, aber sc
  • float hat die möglichen Eigenschaften left, right, none
  • clear hat die möglichen Eigenschaften left, right, both
  • Problem: Wenn alle Elemente in einem Container floaten, hat dieser die Höhe 0

Lösungen für das Collapse Problem

  • Die Empty Div Method: Am Ende der Box ein <div style="clear: both;">
  • Die Overflow Method: Übergeordnete Container bekommt overflow:show/auto
  • Die Pseudo Element Method: clear:both in einem :after Element, verstehen wir später genauer, was das ist.

Was passiert, wenn Dinge zu groß werden

  • Kontrolliert man mit dem CSS-Attribut overflow
  • Die Default-Eigenschaft ist visible
  • Setzt man so von Hand ganz selten, außer man überschreibt die Eigenschaft von wo anders

Was passiert, wenn Dinge zu groß werden

  • Kontrolliert man mit dem CSS-Attribut overflow
  • Will man Dinge einfach abschneiden so nutzt man hidden

Was passiert, wenn Dinge zu groß werden

  • Kontrolliert man mit dem CSS-Attribut overflow
  • Will man den User scrollen lassen dass nimmt man scroll
  • Man erhält Scrollbars, selbst wenn sie gar nicht nötig sind
  • Wichtig: Scrollbars sehen in allen Browsern unterschiedlich aus

Was passiert, wenn Dinge zu groß werden

  • Kontrolliert man mit dem CSS-Attribut overflow
  • Will man den Browser entscheiden lassen: auto
  • Man erhält Scrollbars nur wenn diese nötig sind

Was passiert, wenn Dinge zu groß werden

Auch Text kann zu groß werden

Ein Test Text mit Punkten
  • Hierfür muss ein Kombination von Attributen gesetzt werden
  • Die eigentliche Punkte erzeugt text-overflow:ellipsis
  • Funktioniert aber nur wenn die Box overflow:hidden besitzt
  • Außerdem darf der Text nicht umbrechen white-space:nowrap

Pseudoelemente & Pseudoklassen

  • Man unterscheidet zwischen:
  • Pseudoklasse: Eine CSS-Klasse die implizit verfügbar ist
  • Pseudoelement: Ein HTML-Element, welches im CSS lebt

Pseudoklassen für Interaktionselemente

  • Der klassischte aller Fälle :hover
  • :active betrifft akive Elemente, besonders wichtig im mobilen Anwendungen
  • :focus wichtig für die Bedienung mit der Tastatur und für das Thema Barrierefreiheit

Weitere Pseudoklassen

  • :first-child, :last-child Besonders häufig im Kontext von Listen
  • :nth-child() Häufigster Fall sind Tabellen
  • Weitere Pseudoklassen findet man hier

Spezifizität

  • Schwieriges Wort, im englischen specificity
  • Mehrere Regeln treffen auf das selbe HTML-Element zu, welche Regel gewinnt?
  • Lösung: Ein einfaches Punktesystem
  • Typselector p: 1 Punkt
  • Klasse .box: 10 Punkte
  • Pseudoklasse :visited: 10 Punkte
  • ID #special-box: 100 Punkte
  • Attribute-style <p style="">: 1000 Punkte
  • Der Erfindung des Teufels !important: 100000000000000 Punkte
